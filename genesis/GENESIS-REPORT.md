## Assessment Highlights

### The ONE Sentence
Fabro/Raspberry turns planning documents into autonomously-executing, multi-agent code delivery pipelines with adversarial review and direct trunk integration.

### Key Findings

1. **Scale:** 206,345 LOC Rust across 28 crates. 5,036 commits. 21 plans written in 8 days. Solo operator.

2. **The autodev loop works but is inefficient:** 580 cycles, 16.2% dispatch rate, 83% idle with ready work, 0 trunk landings in the measured period. The system can execute — it just wastes most of its cycles.

3. **Error handling debt:** 2,952 `unwrap()` calls in production code. The engine that enforces code quality does not meet its own standards.

4. **The plans are the best artifact:** The 21 ExecPlan files in `plans/` are the highest-quality output in the repo. They follow rigorous living-document discipline. The plans are better maintained than the code.

5. **Provider routing instability:** Model selection has changed repeatedly (gpt-5.4 removed, MiniMax primary, Kimi for review, Opus for synthesis). Centralization is partial.

6. **Web UI disconnected:** `apps/fabro-web` is upstream Fabro web, not Raspberry-native. The web dashboard doesn't show plan matrix or autodev state.

7. **Two zero-test crates:** `fabro-db` (SQLite + migrations) and `fabro-types` (auto-generated) have no tests.

### Demand Assessment
High-conviction single-operator project. Real autonomous execution capability. Zero external demand signal. The narrowest wedge is `fabro synth genesis` → `raspberry autodev` — turning planning docs into landed code.
